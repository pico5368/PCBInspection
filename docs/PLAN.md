# ML 기반 PCB 검사 시스템 구축안 (v2)

## 1. 프로젝트 목표

PCB(PBA) 외관 검사를 자동화하여 검사 품질, 생산성, 추적성을 동시에 확보한다.

### 1.1 단계별 KPI

| 단계 | KPI | 비고 |
|------|-----|------|
| **PoC / 파일럿** | 과검출률 안정화 (라인 수용성 확보) | 초기엔 과검출 허용, 점진 개선 |
| | 반복 재현성 확보 (동일 샘플 → 동일 판정) | GR&R 테스트 통과 |
| **양산** | Escape 최소화 (최우선) | 후공정 연동으로 추적 |
| | False Reject 최적화 (운영 비용 최소화) | 피드백 루프 기반 |
| | 품목 전환 시간 단축 | Recipe 기반 자동 전환 |

---

## 2. 검사 범위 및 한계 정의

### 2.1 검사 항목

| 검사 항목 | 세부 | 접근법 |
|-----------|------|--------|
| 부품 유무 / 오삽 | Missing, Wrong component | Reference Comparison |
| 부품 극성 | Polarity error | Reference Comparison |
| 위치 / 회전 | Offset, Rotation | Template Matching + 기하 규칙 |
| 납땜 품질 | Cold solder, Insufficient, Excess, Bridge | Anomaly Detection |
| 납볼 / 이물 | Solder ball, Foreign material | Segmentation / Blob Detection |
| 부품 들림 | Tombstone | Anomaly Detection + Rule (그림자 분석) |

### 2.2 명시적 한계 (사전 합의 필수)

- BGA 하부 솔더 → **검사 불가** (X-ray 필요)
- 내부 void → **검사 불가** (X-ray 필요)
- 솔더 높이 정량 측정 → **2D 한계** (3D 프로파일러 필요)

> 프로젝트 시작 전에 품질팀과 반드시 합의

---

## 3. 핵심 설계 원칙

1. **검사 기준서를 먼저 확보하라** — 모델보다 먼저
2. **조명에 투자하라** — 모델 튜닝보다 중요
3. **초기에는 비지도/룰 기반 중심** — YOLO부터 시작하지 마라
4. **시스템 = 모델 + 운영 + UI + 데이터 루프** — 모델만으로 완성 아님
5. **과검출 관리가 1순위** — 라인 신뢰 확보가 먼저

---

## 4. 검사 기준서 (필수 선행 작업)

### 4.1 포함 항목

- 불량 taxonomy (유형 분류 체계)
- 항목별 정의 (수치 기준 포함)
- 허용/비허용 경계 이미지 (사진 기반)
- 경계 사례(borderline) 처리 기준
- 검사 항목별 우선순위

### 4.2 운영 원칙

- **QA 승인 없이는 변경 금지**
- 라벨링 기준과 100% 동일하게 유지
- 변경 이력 관리 (버전 + 승인자 + 사유)

> 검사 기준서 없으면 라벨링 일관성 붕괴 → 모델 학습 품질 저하

## 5. 광학 설계 (DOE 기반)

### 5.1 조명 전략

**단일 조명 사용 금지** → 멀티 조명 구성 필수

| 조명 | 용도 | 대상 결함 |
|------|------|-----------|
| Dome (확산) | 전체 외관, 부품 유무 | Missing, Wrong |
| Low-angle Ring (저각) | 납땜 표면 텍스처, 솔더볼 | Cold solder, Solder ball, Tombstone |
| Coaxial (동축) | 마킹 판독, 패드/브릿지 | Bridge, Polarity |

각 조명으로 동일 위치 순차 촬영 → 검사 항목별 최적 이미지 사용

### 5.2 평가 프로토콜 (DOE)

1. 결함 유형별 실물/인위적 결함 샘플 준비
2. 조명 조합별 촬영
3. Contrast ratio, SNR 정량 측정
4. **재현성 테스트** (시간/환경 변화)
5. 결과 기반 recipe 정의

> 조명 평가에 2~3주 투자 → 모델 튜닝 수개월 절약

## 6. 데이터 전략 (결함 0건 대응)

### 6.1 Phase 기반 접근

| Phase | 시기 | 결함 데이터 | 전략 |
|-------|------|-----------|------|
| 0 | Day 1 | 0건 | 양품 only → Rule + Anomaly Detection |
| 1 | 운영 초기 | 10-50건 | 과검출 수집 → hard negative mining |
| 2 | 데이터 축적 | 100-500건 | 결함 유형별 분류 모델 추가 |
| 3 | 안정기 | 축적 | Supervised 모델 추가 검토, A/B 테스트 |

### 6.2 데이터 정책

- 시간/로트별 정상 데이터 분산 확보 (편향 방지)
- Golden set 별도 관리 (물리 + 디지털)
- 오퍼레이터 피드백 자동 수집
- 합성 데이터 보조 (CutPaste, DRAEM, Defect injection) — 실 데이터 대체 아님

### 6.3 라벨링 전략

- **오퍼레이터**: 과검출 이미지 "정상/결함" 2택 (최소 부담)
- **엔지니어**: 주 1회 축적 데이터 검토, 결함 유형 세분류
- **도구**: Label Studio 또는 자체 웹 UI
- **핵심**: 검사 기준 정의서와 100% 일치

## 7. 모델 전략 (선택지 기반, 고정 금지)

> 모델은 사전 확정하지 않는다. 후보군을 벤치마크하고, ROI 품질/조명 안정성에 따라 결정한다.

### 7.1 부품 비교 / 극성 / 오삽

**후보군** (벤치마크 후 선택):

| 후보 | 장점 | 단점 |
|------|------|------|
| Template matching (classical) | 단순, 빠름, 해석 용이 | 조명/각도 변동에 취약 |
| Feature matching (DINOv2 계열) | 사전학습 활용, fine-tuning 최소 | PCB 부품의 uniform 외관에서 변별력 미지수, 추론 비용 |
| Siamese network | 유사도 학습에 특화 | 학습 데이터 필요 |
| Keypoint 기반 매칭 | 기하학적 정밀도 | 텍스처 빈약한 부품에 약함 |

**결정 시점**: 정합 + 조명 안정화 후

### 7.2 위치 / 회전

- Template matching + 기하 규칙 (우선)
- 필요 시 ML 보조

### 7.3 납땜 품질

| 시기 | 접근 | 후보 |
|------|------|------|
| 초기 | Anomaly Detection (양품 only) | EfficientAD / PatchCore / PaDiM |
| 중기 | 결함 데이터 축적 후 supervised 추가 | Classification head |

**EfficientAD 우선 검토 이유**: PatchCore 대비 추론 속도 ~10x, anomalib 구현 완비

**조건**: 정합 정밀도가 높아야 유효 (정합 오차 → 양품도 anomaly로 판정)

### 7.4 납볼 / 이물

| 후보 | 장점 | 비고 |
|------|------|------|
| Classical blob detection | 단순, 빠름 | 1차 필터 |
| Lightweight segmentation (U-Net, YOLO-seg) | 미세 결함 커버 | 학습 데이터 필요 |
| ML 후단 refinement | blob + ML 조합 | 단계적 추가 |

> SAM류는 필요 시 검토 (기본 아님)

## 8. 파이프라인 구조

```
이미지 취득 (멀티조명 2-4종 순차 촬영)
       │
       ▼
전처리: 정렬 (Registration)
  - Fiducial marker 검출
  - 기준 데이터 기반 affine transform
       │
       ▼
ROI 분할
  - CAD 데이터 → 부품 좌표/패드 좌표 자동 추출
  - 부품별 ROI 자동 생성
  - 수동 보정 UI 제공
       │
       ▼
병렬 검사 (ROI 배치 처리)
  ├─ [부품 유무/극성] → Reference Comparison
  ├─ [위치 오차]      → Template Matching + Rule
  ├─ [납땜 품질]      → Anomaly Detection
  └─ [이물/납볼]      → Blob Detection / Segmentation
       │
       ▼
판정 통합
  - 규칙 엔진: 항목별 임계값 + 가중치
  - 부품별 판정 → 기판 전체 OK/NG
       │
       ▼
결과 출력
  - 결함 위치 오버레이
  - PLC 연동 (OK/NG 신호)
  - 오퍼레이터 피드백 수집
  - 이미지 + 판정 결과 DB 저장
```

### 8.1 예외 처리

| 상황 | 동작 | 비고 |
|------|------|------|
| 정합 실패 | NG 판정 + 알림 | 오퍼레이터 확인 후 재검사 가능 |
| 이미지 취득 실패 | 재촬영 1회 → 실패 시 라인 정지 | 자동 재시도 횟수 제한 |
| 모델 추론 에러 | NG 판정 + 엔지니어 알림 | Fail-safe: 의심 시 NG |
| 조명 이상 | 밝기 기준 이탈 시 경고 → 임계 초과 시 정지 | 기준 이미지 대비 모니터링 |

> 원칙: **판단 불가 시 NG 판정** (Escape보다 과검출이 안전)

## 9. CAD 데이터 연동

### 9.1 데이터 소스 (우선순위)

| 소스 | 장점 | 비고 |
|------|------|------|
| **Pick & Place (CPL)** | 부품 좌표/타입/회전 직접 제공 | 가장 실용적 |
| **ODB++ / IPC-2581** | 레이어+부품+패드 통합 | 지원 시 최적 |
| **Gerber + BOM** | 범용 | 부품 위치 직접 추출 어려움, 파싱 복잡 |

### 9.2 신규 제품 등록 파이프라인

```
CAD 데이터 업로드 (CPL/ODB++/Gerber)
  → 부품 좌표/패드/타입 파싱
  → ROI 자동 생성 + 수동 보정 UI
  → Golden image 촬영 (최소 30장, 시간/로트 분산)
  → Anomaly Detection normal distribution 구축
  → 검사 항목별 임계값 초기 설정 (부품타입별 프리셋)
```

### 9.3 제품 등록 소요 시간

| 유형 | 목표 |
|------|------|
| 단순 제품 (부품 50개 이하) | 수시간 |
| 복잡 제품 (부품 200+) | 0.5~2일 |

> "검사 가능" 정의: 과검출률 ~10% 이하로 동작 가능한 수준

## 10. 다품종 대응 전략

### 10.1 운영 방식

- **Recipe 기반**: 제품별 ROI, 임계값, 조명 설정 분리
- **모델 공유**: 모델 자체는 가능한 공유, 파라미터만 제품별 분리
- **부품타입 재활용**: 동일 타입 (예: 0603 칩저항)은 feature bank 공유

### 10.2 제품 전환

- 바코드/QR 스캔 → 자동 Recipe 전환
- 전환 후 Golden image 1장 검증 촬영 → 정합 정상 확인

## 11. 오퍼레이터 UI 및 피드백 루프

### 11.1 UI 필수 기능

- NG 위치 하이라이트 (원본 위 오버레이 + 색상 코드)
- ROI 확대/축소
- 기준(Golden) 이미지와 나란히 비교
- **1-click 판정 수정** (과검출 시 "정상" 버튼, 2초 이내)
- 코멘트 입력 (결함 유형 보충)
- 통계 대시보드: 시간대별 검출 수, 과검출률, 라인 가동률

### 11.2 피드백 루프

```
판정 "결함" → 오퍼레이터 확인
  ├─ 실제 결함  → 결함 DB → 재학습 데이터
  └─ 과검출     → 과검출 DB → Hard negative mining

판정 "양품" → 후공정 결함 발견 시
  └─ 미검출(Escape) DB → 최우선 재학습 대상
```

> **Escape 데이터가 가장 가치 높다** → 후공정/출하검사와 데이터 연동 필수

## 13. Production Drift 대응

### 13.1 주요 원인

| 원인 | 설명 |
|------|------|
| 기판 로트 차이 | 색상, 재질 편차 |
| 조명 열화 | LED 밝기/색온도 점진 변화 |
| 부품 공급사 변경 | 동일 스펙, 외관 미세 차이 |
| 환경 변동 | 온도, 습도, 주변광 |

### 13.2 대응 전략

- **입력 정규화**: 히스토그램 정규화, 화이트밸런스 보정
- **Score 분포 모니터링**: 정상 판정의 anomaly score 평균/분산 추이 실시간 추적
- **조명 밝기 모니터링**: 기준 이미지 대비 밝기 변화 감지
- **정합 실패율 추적**: drift 간접 지표
- **Golden image 주기적 갱신**: 월 1회 재촬영
- **자동 알림**: 분포가 기준선에서 벗어나면 엔지니어 알림
- **Staged rollout**: 새 모델 → 기존 모델과 병렬 → 비교 후 전환

---

## 14. Golden Sample 관리

- **물리적 관리**: 전용 보관함, 온습도 관리, 취급 절차 문서화
- **열화 대응**: 솔더 표면 산화 → 월 1회 재촬영
- **디지털 Golden**: 양품 N장의 평균 feature를 디지털 기준으로 사용 (물리 의존도 감소)
- **다중 Golden**: 로트 변동 커버 → 3-5개 유지
- **이력 관리**: 교체 시 이전/이후 검사 결과 비교 리포트 자동 생성

---

## 15. 반복 재현성 검증 (GR&R)

### 15.1 테스트 프로토콜

- 동일 샘플 N회 반복 촬영 → 판정 일관성 측정
- 시간 간격 두고 재촬영 (1시간, 4시간, 24시간)
- 환경 변화 조건 (온도, 조명 워밍업 전/후)

### 15.2 합격 기준

- 판정 일치율 > 99% (동일 조건)
- 환경 변화 시 판정 일치율 > 95%

> **양산 전 필수** — GR&R 미통과 시 양산 투입 불가

## 16. Latency 및 시스템 설계

### 16.1 Latency Budget

> 실제 라인 택타임 확인 후 역산 필요. 아래는 택타임 10초 기준 예시.

| 단계 | 시간 | 비고 |
|------|------|------|
| 이미지 취득 (멀티조명 3종) | 2-3초 | 조명 전환 + 트리거 대기 포함 |
| 전처리 (정합, ROI) | 0.5-1초 | |
| 추론 (전체 ROI 배치) | 3-5초 | 부품 200+ 시 빠듯 |
| 판정 + 결과 전송 | 0.5초 | |
| 마진 | 0-1.5초 | |

### 16.2 최적화 전략

- ROI 병렬 배치 추론: 동일 부품 타입 ROI 묶어 GPU 한 번에 투입
- 모델 경량화: EfficientAD-S, MobileNet backbone
- TensorRT FP16: 정확도 손실 미미, 속도 1.5-2x
- **CPU/GPU 파이프라이닝**: 전처리(CPU)와 추론(GPU) 동시 실행
- **End-to-end 택타임 포함 측정** (모델 추론만이 아닌 전체)

### 16.3 하드웨어

| 산업용 GPU 박스 (RTX A2000/A4000) | 성능 우선 |
